# Android Instagram Volatile Memory Forensics 🔍📱
## 📌 Introduction
In my Advanced Digital Forensics course (CYS 405), our team conducted a behavioral analysis of volatile memory artifacts generated by Instagram on an Android emulator under different runtime conditions. This repository contains my contribution to the team project: Python scripts for automated forensic artifact detection and classification from Android logcat, meminfo, and notification dumps.

## 📋 Project Overview
The research examined Instagram's volatile memory behavior across seven runtime scenarios on an Android emulator: immediate acquisition, delayed acquisition, background/idle state, RAM pressure (multitasking), application switching, force-close, and screen lock. This repository focuses on my contribution to the project **the application switching and force-close scenarios**, providing automated analysis of forensic artifacts.

## 🛠️ Technologies Used
Python - Core analysis language

Regex (re module) - Pattern matching for artifact detection

ADB (Android Debug Bridge) - Data acquisition from emulator

Logcat / dumpsys - Source of forensic data

## 💻 Validation Code Overview
The Python scripts App_switch.py and Force_quit.py are designed to automatically detect and classify forensic artifacts from Android outputs after Instagram is switched to another app or force-closed.

## 🎯 My Key Contributions

**Script	Scenario	Purpose:**

## 🔬 Artifact Categories Analyzed
Full/Partial Message Fragments - Plaintext message recovery
Instagram Direct/Thread-Specific Traces - DM-related metadata
Keyboard/Typing Activity - User input behavior
Push/Background Service Traces - Firebase, FBNS, background processes
Notification Artifacts - Notification channels and metadata
Foreground/Background Transitions - App lifecycle events
Memory/Process Traces - RSS, PSS, process persistence
Force-Close Termination Traces - App killing events

## 📊 Analysis Features
Regex-based pattern matching across multiple forensic artifacts
State classification (PRESENT / PARTIAL / MISSING) based on match counts
Timestamp extraction from logcat for timeline reconstruction
Cross-file aggregation combining logcat, meminfo, notification, and activity dumps
